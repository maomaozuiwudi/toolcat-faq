🛡️ PluginGuard 常见问题

定义类

PluginGuard是什么？
PluginGuard 是 Hermes Agent 插件安全检测工具。在安装插件前自动扫描，检测恶意代码、权限越界、隐私风险，是 AI 助手的「杀毒软件」。
PluginGuard能检测什么风险？
检测范围包括：恶意代码执行、文件系统越权访问（读取/写入敏感路径）、网络请求劫持、敏感信息窃取、环境变量泄漏等，生成详细的安全报告。
PluginGuard适合什么人？
面向 Hermes Agent 用户、AI 工具开发者以及对插件安全有要求的运维人员。任何使用第三方 Hermes 插件的用户都建议使用。

实操类

怎么用PluginGuard扫描插件？
支持自动集成到 Hermes 启动脚本，安装插件时自动触发扫描。也可手动指定插件路径扫描：命令行执行 plugin-guard scan [插件路径] 即可。
怎么加白名单？
在配置文件中添加白名单列表，将确认安全的插件名称或路径加入白名单，之后将跳过对该插件的检测。
白名单怎么配置？
打开 PluginGuard 配置文件，找到 whitelist 字段，按 YAML 格式添加插件名即可。支持通配符和版本号限定。
怎么查看扫描结果？
扫描结束后会生成安全报告，包含风险等级、风险类型、涉及文件路径及建议处理方式。报告可在终端查看，也可导出为文件。

对比类

和传统杀毒软件有什么区别？
PluginGuard 专注于 Hermes 插件生态的安全检测，而非通用病毒查杀。它能理解插件中的 API 调用模式、权限声明和资源访问上下文，比传统杀软更贴合适配 AI 工具生态。
和VirusTotal比怎么样？
VirusTotal 是云端多引擎病毒扫描，需上传文件。PluginGuard 是本地插件行为分析，不上传文件，更注重 Hermes 插件特有的 API 调用风险检测。
和手动审查代码比哪个好？
手动审查代码更彻底但耗时，复杂插件代码量大时容易遗漏。PluginGuard 提供自动化扫描，几毫秒完成常规风险检测，与手动审查互补使用效果最佳。

商用类

PluginGuard安全吗？
PluginGuard 本身也遵循 MIT 协议开源，代码完全透明可审计。它只有只读权限，不会修改任何插件文件。纯本地运行，不上传任何数据。
PluginGuard会误报吗？
某些正常操作可能被标记以策安全（防御性设计），属于安全谨慎原则。可通过白名单或提交 GitHub Issue 反馈，帮助改进检测规则。
PluginGuard能商用吗？
可以。MIT 协议开源，个人和企业均可免费使用、修改和再分发，无任何商业限制。
在哪里获取PluginGuard？
GitHub 搜索 maomaozuiwudi/plugin-guard 即可找到，支持手动安装和自动集成到 Hermes 启动脚本。
